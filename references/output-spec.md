# 输出规范

## 语言要求

所有面向用户的预览图、最终 PNG、PDF、命令行提示、警告和报错必须使用简体中文。MARD 色号、命令名、参数名、JSON 字段和输出文件名保持原有英文形式，避免破坏兼容性。

## 预览任务

`preview` 会生成用于选择版本的工作文件：

- 输入图片副本；
- 按目标网格处理后的原图；
- 干净色块版和轻度抖动版预览 PNG；
- 原图/干净版/抖动版三图对比页；
- 可重复生成的 clean/dither 图案 JSON；
- 记录参数、色库哈希、指标、实际图案边界、按 `2.6 mm` 计算的尺寸和底板建议的 `job.json`。

预览文件是工作材料，不属于最终客户交付。

## 实际图案边界

- 在所选版本中查找最小非透明边界。
- 最终渲染只裁掉外围透明留白。
- 内部透明孔洞保留为白色空格。
- 最终施工图坐标从裁剪后图案的 `1` 开始编号。
- 没有非透明格时禁止最终导出。
- 从 `52×52`、`78×78`、`104×104` 中选择能容纳实际宽高的最小底板。
- 预计实体宽高按每颗间距 `2.6 mm` 分别计算。

## 最终交付

成功运行 `finalize` 后只生成：

```text
<variant>-pattern.png
<variant>-pattern.pdf
```

最终目录必须为新目录或空目录，不生成独立用量 CSV、清单文件、额外总览 PNG 或第二份 PDF。

## 完整施工图 PNG

- 每颗非透明拼豆对应一个彩色方格；
- 每个彩色方格中央显示规范 MARD 色号；
- 深色格使用白字，浅色格使用黑字；
- 透明孔洞显示白色空格；
- 上、下、左、右四边均显示坐标；
- 普通格线较细，每 5 格加强、每 10 格进一步加粗；
- 信息区使用简体中文并显示：
  - `MARD拼豆施工图`；
  - `图案尺寸`；
  - `预计成品尺寸`；
  - `建议底板`；
  - `总豆数`；
  - `使用色数`；
  - `适合用途`；
- 小尺寸图案必须自动换行并增加信息区高度，不得让中文超出边界；
- 图例按规范色库顺序显示颜色、色号和准确数量，例如 `A01（18颗）`；
- 图像使用高分辨率方格，便于放大查看。

## 单一 PDF

- 第一页包含完整施工图和图例；
- 根据图纸形状自动选择 A4 纵向或横向；
- 首页页脚使用 `完整图纸 / 第1页`；
- 小图在完整图纸可读时保持单页；
- 密集图案在同一 PDF 中追加 8 毫米可读施工分图；
- 分图使用裁剪后图案的全局坐标；
- 分图标题使用 `分图1,2`、`列1–23，行24–48` 等中文格式；
- 相邻关系使用 `相邻页面：上、下、左、右`，无相邻页时显示 `无相邻页面`；
- 页脚使用 `8毫米可读施工分图` 和 `PDF第2页` 等中文格式；
- PDF 标题、主题等元数据使用中文；
- PDF 中文必须使用可嵌入且能被文本提取工具识别的系统中文字体。

8 毫米仅为打印可读格尺寸，实体拼豆尺寸仍按每颗间距 `2.6 mm` 计算。

## 准确数量

全部图例数量之和必须等于非透明拼豆总数；透明格不计入。图例顺序遵循规范色库顺序。

## 大图确认

符合任一条件时，最终导出必须等待明确确认：

- 非透明拼豆超过 40,000 颗；
- 单一合并 PDF 预计超过 50 页。

确认后使用 `--yes-large` 重新运行。
